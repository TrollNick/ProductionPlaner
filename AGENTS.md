# Product principles for TAKT

TAKT is a shared reality map for a small, flexible production team. It must never become a scheduling prison.

- Show consequences, conflicts, dependencies and uncertainty clearly, but keep manual decisions possible.
- Never silently move committed dates. Automatic behavior must be visible and individually overridable.
- Delays may push dependent work back. Early completion must only pull later work forward when the user explicitly enables it.
- Preserve the original baseline when reality changes. Record the current date, the type of deviation and its human-readable reason separately.
- Plan work in German business days, Monday through Friday. Do not count weekends as production days.
- Prefer a dense overview that answers: What is missing? What can start? What changed? When can we deliver?
- Timeline and calendar are two views of the same data and must always agree.
- Keep the product lightweight. Do not introduce ERP, inventory, time tracking or automatic resource optimization unless explicitly requested.
